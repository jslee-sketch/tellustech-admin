# TELLUSTECH ERP — Claude Code 개발 시작 가이드 (최종)

## 프로젝트 개요
- **회사**: Tellustech Vina (TV) + Vietrental (VR) — 2개 법인, 1개 ERP
- **위치**: 베트남 박닌 본사, 5개 거점 (BN/HN/HCM/NT/DN)
- **사업부**: IT Rental(복합기) / T&M Repair / T&M Calibration
- **사용자**: ~30명 내부직원 + 고객 포탈
- **교체 대상**: ECOUNT ERP (더 이상 사용 안 함)
- **유지**: MISA (베트남 공인회계) — 엑셀 반자동 연계

## 기술 스택
- **Frontend**: Next.js (App Router) + TypeScript + Tailwind CSS
- **Backend**: Next.js API Routes (또는 NestJS)
- **DB**: PostgreSQL
- **ORM**: Prisma
- **인증**: JWT + RBAC (로그인 시 회사코드 선택 — ECOUNT 방식)
- **실시간**: WebSocket (채팅 + 알림)
- **파일저장**: AWS S3 또는 로컬
- **바코드**: QuaggaJS (브라우저 카메라)
- **전자서명**: HTML5 Canvas
- **SNMP**: net-snmp (Node.js) — Samsung SL-X7500, Sindoh D410/D320/D330
- **AI번역**: Claude API (저장시점 번역, 채팅 실시간 번역)
- **엑셀**: ExcelJS / SheetJS (업로드/다운로드 + 대량 데이터 입력)
- **PDF압축**: Ghostscript 또는 pdf-lib (업로드 시 자동 50~80% 절감)
- **지도**: Google Maps API (AS 출동 거리 자동계산)
- **OCR**: 차량 미터기 사진 → 숫자 파싱
- **UI**: 다크모드 기본

## 10개 모듈 구조
1. **기초등록** — 부서/직원/거래처(CRM 6탭)/품목/프로젝트/창고/일정관리(CFM)/라이선스관리
2. **렌탈관리** — IT(복합기 기본료+초과과금) + TM(계측기 매출→매입 S/N별 관리) + 자동오더생성
3. **재고관리** — 지점별 재고, 자산 감가상각, 바코드 스캔 입출고 (별도 입력폼 없음)
4. **교정관리** — 교정이행(복수장비세트+엑셀업로드), 성적서PDF(압축), 연간알림(11개월), 고객다운로드
5. **AS관리** — 접수~완료 → 목록에서 출동버튼 → 출동등록(Google Maps+미터기OCR+사진), 미수금차단
6. **매출관리** — 프로젝트코드별 분류, S/N재고확인, 복수품목세트+엑셀업로드, 청구서PDF
7. **매입관리** — 공급사별, PO관리, S/N재고확인, 복수품목세트+엑셀업로드, 창고입고 자동연동
8. **인사관리** — 사건기반수시평가, 정기인사평가(5단계), AI자동평가(편향성분석), 입사카드(전자서명+PDF), 퇴사카드(체크리스트+PDF), 연차, 급여, 인센티브
9. **IT계약관리** — 복수장비+과금조건(S/N 수백개+엑셀업로드), 계약/기술/재경 담당자, 보증금/설치비/배송비, 계약PDF, 만료알림, 매출자동반영
10. **재경관리** — 비용관리(매입/매출/일반, 원가배분), 미지급/미수금관리(자동생성→선택→업데이트, 지연사유이력)

## 핵심 아키텍처

### 회사코드 (Multi-Company)
- 로그인 시 회사코드 선택: TV (Tellustech Vina) / VR (Vietrental) — ECOUNT 방식
- 세션 전체에 company_code 적용, 모든 쿼리에 WHERE company_code = :session 자동
- **공유** (company_code 없음): 거래처(clients), 품목(items), 창고(warehouses) — 마스터 데이터만
- **완전 분리** (company_code 필수): 직원, 부서, IT계약, 렌탈, AS, 교정, 매출, 매입, 급여, 인센티브, 재경, 일정, 라이선스 — 그 외 전부
- 일반 직원: 자기 회사코드로만 로그인
- 관리자(대표): TV/VR 선택 로그인 + 통합 조회 모드 전환 가능
- 고객 포탈: 회사코드 불필요 (거래처는 공유 데이터)

### S/N 기반 아키텍처
- S/N(Serial Number)이 모든 모듈의 통합 기준키
- S/N 재고확인 정책:
  - **IT렌탈 = 엄격 (필수)**: 자사 재고에 있어야만 등록 가능
  - **나머지 (TM렌탈/교정/AS/매출/매입) = 느슨 (확인만)**: 외부 렌탈 등 재고에 없을 수 있으므로 확인만 하고 진행

### 자동 코드 생성
모든 등록 화면에서 코드/번호 자동생성:
- 거래처코드: CL-날짜-순번
- 품목코드: ITM-날짜-순번
- 사원코드: TNV/VNV 체계 유지
- 계약번호: VRT/TLS 체계
- AS전표번호: YY/MM/DD-순번 (26/04/21-01)
- TM렌탈: TM-날짜-순번
- 교정: 자동
- 평가코드: INC-/EVAL- 날짜-순번
- 입사카드: ONB-날짜-순번
- 퇴사카드: OFF-날짜-순번
- 연차신청: LV-날짜-순번
- 비용: EXP-날짜-순번
- 일정: SCH-날짜-순번
- 라이선스: LIC-날짜-순번

### 저장시점 AI 번역
- 입력: 베트남어/영어/한국어 아무 언어로 자유 입력
- 저장 시: 원문 보존 + Claude API가 나머지 2개 언어 자동 번역
- DB 저장: content_vi, content_en, content_ko 3개 필드
- 조회 시: 사용자 설정 언어로 즉시 표시 + 원문보기 버튼
- 번역 수정: 관리자 수동 수정 가능
- 적용 범위: AS내역, 사건평가, 고객요청, 메모, 채팅 등 자유서술 필드 전체

### 매출 자동반영 시스템
- IT계약, TM렌탈, 교정이행 화면에 "매출 자동반영" 버튼
- 클릭 시 매출전표에 맞는 값을 가져와서 매출로 자동 반영
- 매출/매입 저장 시 미수금/미지급금 자동 생성

### 대량 데이터 입력
- IT계약 S/N: 수백 개 등록 가능 + 엑셀 업로드
- TM렌탈 품목세트: 수백 개 추가 가능 + 엑셀 업로드
- 교정 장비세트: 수백 개 추가 가능 + 엑셀 업로드
- 매출/매입 품목행: 수백 개 추가 가능 + 엑셀 업로드

### AS → 출동 연결 흐름
- AS 접수 → AS 목록에 등록됨
- AS 목록에서 해당 건 옆에 "출동" 버튼
- 출동 버튼 클릭 → 출동등록 화면으로 이동 (AS전표 자동 연결)
- 출동 등록: 출동수단 선택 → Google 거리 자동표시 → 미터기 사진+OCR → 거리 자동비교

### 입출고 (바코드 스캔)
- 별도 입력폼 없음 — 모바일 웹브라우저에서 카메라로 바코드 스캔
- 스캔 → 입고/출고 선택 → 사유 선택 (매입/교정/수리/렌탈/데모/회수/소모품출고)
- 해당 창고에 재고 자동 반영

## AI 평가 시스템 상세

### 사건기반 수시평가
- 누구나 수시로 작성 (작성자/대상자 지정)
- 칭찬 또는 개선필요 유형 선택
- 구체적 사건 서술 (50자 이상)
- 대상자 본인에게 비공개, 상위 평가자만 열람
- AI가 사건 분석 시 평가자와 피평가자를 동시 평가 (편향성/성격패턴)

### 정기 인사평가
- 관리자가 시작 (평가번호 자동생성)
- 평가자/피평가자 선택
- 질문은 사전 등록 가능
- 5단계 답변: 매우잘함(10) / 잘함(8) / 보통(6) / 노력요함(4) / 많은노력필요(2)
- 질문수 × 답변 합산 → 100점 만점 자동환산
- 마감일 설정, 제출일 자동기록

### AI 종합평가
AI가 3가지를 종합 분석:
1. **사건기반 수시평가** — 사건 내용 분석, 편향성 감지 (특정인 선호/비선호, 점수 치우침)
2. **정기인사평가** — 평가자의 점수 패턴 분석 (지나치게 높게/낮게, 특정인 편향 위험도)
3. **ERP 데이터** — AS TAT(15%), 출동효율(10%), 매출기여(15%), ERP입력속도(5%), ERP마감준수(10%), ERP숙련도(5%), 근태(5%)

공정성 보장:
- 가중평균 (관리자 > 동료 가중치)
- 치우침 방지 (모든 대상에 만점/최하점만 주는 평가자 자동 제외, ±2σ)
- 올림픽 방식 (최고점+최저점 제거 후 평균)
- 부서별 보정
- 익명성 보장
- AI가 편향성을 감지하여 점수 자동 보정

결과: 항목별 점수+이유 + 종합점수+이유 + A/B/C/D등급 + 월별추이 + 관리자 ±10% 조정(사유필수)

## ERP 내장 채팅 시스템
- WebSocket 기반 실시간 채팅
- 메시지 전송 시 Claude API로 나머지 2개 언어 자동번역 → 3개 언어 DB 저장
- **수신자 표시 설정** (사용자 선택):
  - 1개 언어: 내 설정 언어만 (예: 🇰🇷 용지 걸림)
  - 2개 언어: 번역 + 원문 (예: 🇰🇷 용지 걸림 + 🇻🇳 Máy bị kẹt giấy)
  - 3개 언어: 모두 표시 (🇻🇳 + 🇬🇧 + 🇰🇷)
- 원문 언어 자동감지 + 국기 아이콘 표시
- AS전표번호/계약번호 멘션 시 해당 건으로 바로 링크
- 1:1 채팅 + 그룹 채팅 (부서/프로젝트별)
- 읽음 확인, 파일/사진 첨부

## 21개 입력화면 구성

### 시스템
1. **로그인** — 회사코드(TV/VR) + 아이디 + 비밀번호 + 언어

### 기초등록
2. **거래처 등록** — 코드자동 + 기본정보 + 계좌/결제 + 영업(유입경로/소개자/등급) + 마케팅
3. **품목 등록** — 코드자동, 바코드없음, 자산정보없음 (품목≠재고), 상품/소모품/부품 구분
4. **직원 등록** — 코드자동 + 기본 + 신분증 + 급여 + 계약 + 증명사진
5. **일정(마감) 등록** — 코드자동 + 반복주기 + 마감일시 + 사전알림 + 대상직원 + 보고대상 + 연관모듈
6. **라이선스 등록** — 코드자동 + 취득/만료/알림 + 담당자 + 갱신비용 + 증서첨부

### 렌탈
7. **IT계약 등록** — 회사없음 + 거래처 + 설치주소(상단) + 보증금/설치비/배송비/부가서비스 + 계약/기술/재경 담당자(이름/HP/Office/이메일) + 장비+과금조건 복수세트(S/N 수백개+엑셀업로드) + 매출자동반영
8. **TM렌탈 등록** — 고객정보(계약번호+거래처+주소+담당자3종) → 매출정보 먼저(품목/옵션/SN/기간/매출단가 세트 수백개+엑셀) → S/N별 매입정보(매입처/매입금액/커미션) → 이익자동계산 + 매출자동반영
9. **IT부문 월별 사용량 컨펌** — 카운터입력(SNMP/수동/사진) + 청구자동계산 + 고객전자서명

### 교정
10. **교정이행 등록** — 회사없음 + 고객정보(담당자3종) + 장비세트(장비명/번호/SN/기준금액/실제금액/성적서번호, 수백개+엑셀) + 수량없음 + 매출자동반영

### AS
11. **AS접수** — 회사없음 + 전표번호자동 + 거래처 + 장비/SN(느슨확인) + 증상(다국어자동번역) + 사진 + 미수금차단 자동확인
12. **출동등록** — AS목록에서 출동버튼으로 진입 + 출동수단 + Google거리 + 미터기사진OCR + 거리자동비교 + 교통비/영수증

### 매출매입
13. **매출전표** — 회사없음 + 영업담당 추가 + 사용기간(optional) + 품목세트 수백개+엑셀 + SN느슨확인 + 미수금자동생성
14. **매입전표** — 회사없음 + 영업담당 추가 + 사용기간(optional) + 품목세트 수백개+엑셀 + SN느슨확인 + 미지급자동생성

### 인사
15. **사건평가** — 코드자동 + 작성자자동 + 대상자 + 칭찬/개선 + 50자이상 + 증빙 + 공개범위 (AS전표/계약번호 연결 없음)
16. **정기평가** — 코드자동 + 평가자/피평가자 + 5단계답변 + 100점환산 + AI편향성분석 + 관리자최종배점
17. **입사카드** — 코드자동 + 학력/경력 + 동의서전자서명 + 증명사진 + PDF자동생성
18. **퇴사카드** — 코드자동 + 반납/지급/중지/발급 체크리스트 + 동의서서명 + PDF자동생성
19. **연차신청** — 코드자동 + 잔여연차표시 + 휴가유형 + 승인자

### 재경
20. **비용등록** — 코드자동, 회사없음 + 비용구분(매입/매출/일반) + 관련전표연결 + 원가배분(수량/금액 선택)
21. **미수미지급** — 매출/매입/비용에서 자동생성된 목록 → 건 선택 → 상단자동채움 → 입금확인+지연사유만 계속 업데이트
