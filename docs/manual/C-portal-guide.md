# 환영합니다

Tellustech 의 IT 렌탈·계측기 렌탈·정비 서비스를 이용해 주셔서 감사합니다.

**고객 포탈**은 다음 4가지를 한 곳에서 처리할 수 있는 온라인 창구입니다.

1. **AS 요청** — 본인 보유 장비 고장 신고
2. **소모품 요청** — 토너·잉크·부품 등 호환 품목 주문
3. **사용량 컨펌** — 월별 청구 사용량 확인 후 모바일 서명
4. **교정 성적서** — 검교정 PDF 다운로드

PC·태블릿·스마트폰 어느 화면에서나 동일하게 작동하며, 한국어 / 베트남어 / 영어 3개 언어를 지원합니다.

> 본 가이드는 **고객사 담당자**가 포탈을 사용할 때 참고합니다. 사내 직원용 ERP 매뉴얼(별책 A)과는 다른 별도 문서입니다.

# 1. 시작하기 — 접속과 로그인

## 1.1 포탈 접속

브라우저(Chrome / Safari / Edge)에서 Tellustech 담당자가 안내한 포탈 주소로 접속합니다.

```
https://portal.tellustech.co.kr
```

> 정확한 도메인은 담당자에게 확인하세요. 사내 직원용 도메인과 분리되어 있습니다.

## 1.2 로그인 입력값 — 어디로 흘러가는가

| 입력 항목 | 예시 | **이 값이 시스템에서 하는 일** |
|---|---|---|
| **고객코드** | `CL-260101-001` | DB 의 `Client.clientCode` 와 매칭 → 본인 거래처 ID 가 세션에 고정. 이후 모든 화면이 이 ID 로 데이터 필터 |
| **비밀번호** | (담당자 발급) | bcrypt 해시 비교. 일치 시 JWT 발급, 불일치 5회면 일시 잠금 |
| **언어** | 한국어 / Tiếng Việt / English | `User.language` 에 저장 → 모든 화면 라벨·메시지에 즉시 적용. 데이터 자체는 3언어 컬럼에 함께 저장됨 |

**입력 결과:**
- 로그인 성공 → `JWT` 쿠키(`tts_session`) 발급, `/portal` 메인으로 이동.
- 로그인 후 표시되는 모든 데이터(IT 계약·요청 현황 등)는 자동으로 본인 거래처 ID 로 필터링됩니다. 다른 회사 데이터는 절대 노출되지 않습니다.
- 비밀번호 5회 오답 시 계정 일시 잠금 (담당자 해제 필요).

> **회사코드(TV/VR) 입력은 필요 없습니다.** 귀사가 거래하는 Tellustech 법인이 TV 인지 VR 인지는 고객코드 자체에 이미 연결되어 있어 자동 라우팅됩니다.

## 1.3 로그인 화면 진입

로그인 페이지에서 화면 하단의 **🛒 고객 포탈 로그인 / Customer Portal** 버튼을 누르거나, 안내된 URL 에 `?portal=1` 파라미터가 붙어 있으면 자동으로 포탈 모드 폼이 표시됩니다 (회사코드 입력란이 사라지고 「고객코드」로 변경).

## 1.4 비밀번호 분실

비밀번호를 잊으신 경우 Tellustech 담당자에게 연락해 재설정을 요청해 주세요. 보안상 자가 재설정은 제공하지 않습니다 (포탈에서는 이메일 기반 재설정 링크를 발송하지 않습니다 — 사칭 방지 목적).

# 2. 메인 화면 한눈에 보기

로그인 직후 표시되는 **메인 화면**은 4개 영역으로 구성됩니다.

## 2.1 헤더 — 회사명 / 고객코드

화면 상단에 귀사의 회사명과 고객코드(`CL-...`)가 표시됩니다.
**미수금 정지 상태**(상환 지연 등)인 경우 빨간색 `정지 / Bị khóa` 배지가 표시되며, AS·소모품 신규 요청이 일시적으로 비활성화됩니다. 해제는 결제 확인 후 자동 처리되거나 담당자에게 문의해 주세요.

## 2.2 빠른 요청 카드 — 4개

- 🛠 **AS 요청 / Yêu cầu BH**
- 📦 **소모품 요청 / Yêu cầu vật tư**
- ✍️ **사용량 컨펌 / Xác nhận sử dụng**
- 📄 **교정 성적서 / Chứng chỉ** (괄호 안 숫자는 다운로드 가능 건수)

각 항목을 누르면 해당 화면으로 이동합니다.

## 2.3 내 IT 계약 카드

활성/임박 IT 계약 최근 10건을 계약번호 순으로 표시합니다. 각 줄에는 **계약번호**, **상태**(`ACTIVE` / `DRAFT` 등), **장비 수**가 함께 보입니다.

## 2.4 내 요청 현황 표

직전에 등록한 AS·소모품 요청 최근 20건의 진행 상황입니다. 4단계로 흐릅니다.

| 단계 | DB 값 | 의미 | 누가 바꾸는가 |
|---|---|---|---|
| **요청 / Yêu cầu** | `RECEIVED` | 접수 완료, 담당자 배정 대기 | 자동 (등록 시점) |
| **진행중 / Đang xử lý** | `IN_PROGRESS` | 담당자 배정·작업 진행 중 | Tellustech 담당자가 변경 |
| **완료 / Hoàn tất** | `COMPLETED` | 작업 완료, 고객 확인 대기 | Tellustech 담당자가 디스패치 완료 + 서명 |
| **확인됨 / Đã xác nhận** | `CONFIRMED` | 고객이 확인 → 종결 | **귀사**가 「확인」 버튼 클릭 |

### 「확인」 버튼이 하는 일

상태가 `COMPLETED` 인 줄에만 우측에 **확인** 버튼이 활성화됩니다. 이 버튼을 누르면:

1. API `POST /api/portal/tickets/{id}/confirm` 호출.
2. 서버 검증: 본인 거래처의 티켓인가? 현재 상태가 `COMPLETED` 인가?
3. 통과 시 → `status = CONFIRMED`, `confirmedAt = 지금` 으로 업데이트 → 작업 종결.
4. 페이지가 자동 새로고침되어 줄이 「확인됨」 으로 갱신.

> 한 번 확인하면 되돌릴 수 없습니다. 작업 결과에 이의가 있으면 **확인 버튼을 누르기 전에** Tellustech 담당자에게 연락해 주세요.

# 3. AS 요청

장비가 고장났거나 점검이 필요할 때 사용합니다.

## 3.1 입력값별 데이터 영향도

| 입력 항목 | 출처 / 값 | **이 값이 시스템에서 하는 일** |
|---|---|---|
| **장비 SN** (드롭다운) | `/api/portal/my-equipment` 가 반환하는 본인 활성 장비 목록 | 선택한 SN 이 `AsTicket.serialNumber` 에 저장 → Tellustech 담당자가 그 장비의 계약·이력·교정 기록을 즉시 조회 |
| **원문 언어** (VI/KO/EN) | 라디오 / 셀렉트 | `AsTicket.originalLang` 에 저장. 자동 번역 시 어느 언어를 원본으로 볼지 시스템이 판단하는 기준 |
| **증상 (VI/KO/EN)** 3칸 중 1칸 | 자유서술 텍스트 | 입력 즉시 저장 시 Claude API 가 나머지 2개 언어로 자동 번역 → `symptomVi` / `symptomEn` / `symptomKo` 3컬럼에 모두 채움. 결과: 한국어로 쓴 내용을 베트남 담당자가 베트남어로, 영국 본사가 영어로 즉시 읽음 |

**드롭다운에 표시되는 형식:**
```
SN-12345 · ITM-260101-001 LaserPrinter X7500 (TLS-260101-001)
```
순서대로 **시리얼 번호 · 품목코드 품목명 (계약번호)** 입니다.

**드롭다운에 어떤 SN 이 보이는가:**
- 활성 IT 계약(`itContractEquipment.removedAt = null`)에 등록된 장비 + 종료 전 TM 렌탈(`tmRentalItem.endDate >= 오늘`)의 장비.
- 따라서 회수·교체된 장비는 자동으로 사라지고, 신규 등록된 장비는 자동으로 보입니다 (페이지 새로고침 후).

## 3.2 입력값 검증 규칙 (저장 시점)

| 검증 | 실패하면 |
|---|---|
| 증상 3개 칸 중 **최소 1개**에 텍스트가 있어야 함 | `invalid_input` 에러, 「증상을 입력하세요」 표시 |
| 본인 거래처 세션 (역할=CLIENT) | `client_only` 에러 — 정상 흐름에서는 발생하지 않음 |
| 자유 입력 SN 도 허용 | 드롭다운에 없는 SN 도 그대로 저장됨 (외부에서 가져온 장비 등) |

## 3.3 접수 후 일어나는 일

**AS 접수 / Tiếp nhận BH** 버튼 클릭 시 시스템이 다음을 한 번에 수행합니다.

1. **티켓번호 자동 발급** — `YY/MM/DD-NN` 형식 (예: `26/04/27-01`) — 그 날의 일련번호.
2. **티켓 생성** — `kind = AS_REQUEST`, `status = RECEIVED`, 접수일시·본인 거래처 ID 자동 채움.
3. **3언어 번역 보관** — 입력하지 않은 두 언어 칸이 자동 번역되어 저장.
4. **사내 알림** — Tellustech AS 담당자에게 즉시 통지 (사내 ERP 「AS 접수」 화면에 빨간 미읽음 배지로 표시).
5. **고객 알림** — 등록된 이메일 또는 카카오톡으로 「접수 완료 — 번호 26/04/27-01」 메시지 전송.
6. **메인 화면 갱신** — 「내 요청 현황」에 즉시 새 줄로 표시.

## 3.4 미수금 정지(BLOCKED) 시 차단 동작

귀사 거래처가 `BLOCKED` 상태이면:
- 「AS 요청」 빠른 카드가 회색 + 취소선으로 표시.
- 강제로 URL 직접 입력 시 API 가 `403 receivable_blocked` 로 거절.
- 결제 확인되면 거래처 상태가 `WARNING` 또는 `NORMAL` 로 자동 복귀 → 메뉴도 즉시 활성화.

# 4. 소모품 요청

토너·잉크·부품 등 정기 소모품을 요청합니다.

## 4.1 입력값별 데이터 영향도

| 입력 항목 | 출처 / 값 | **이 값이 시스템에서 하는 일** |
|---|---|---|
| **품목** (드롭다운) | `/api/portal/my-supplies` — 본인 활성 장비의 호환 매핑(`ItemCompatibility`) 결과 | 선택한 itemId 가 라인에 저장. 호환 화이트리스트에 없는 itemId 는 서버가 `not_compatible` 로 거절 |
| **수량** | 1 이상 정수 | 라인의 `quantity` 에 저장. Tellustech 출고 담당자가 이 값으로 발주·창고 출고 처리 |
| **메모** (선택) | 자유 텍스트 | 라인의 `note` 에 저장. 색상·모델·긴급 여부 등을 담당자에게 직접 전달 |

## 4.2 호환 매핑이 어떻게 결정되는가

드롭다운에 보이는 품목은 다음 SQL 과 동등한 흐름으로 계산됩니다.

```
본인 활성 장비의 itemId 집합 (PRODUCT)
   ↓
ItemCompatibility 테이블의 (productItemId, consumableItemId) 매핑
   ↓
consumableItemId 집합 = 드롭다운 표시 후보
```

따라서:
- **신규 장비가 등록**되어도 호환 매핑이 같이 등록되지 않으면 새 소모품이 안 보일 수 있습니다 → 담당자에게 매핑 등록 요청.
- **장비가 회수**되면 그 장비 전용 소모품은 다른 장비와 공통이 아닐 경우 자동으로 드롭다운에서 사라집니다.
- 호환 매핑은 관리자 화면(`/admin/item-compatibility`)에서 Tellustech 가 직접 관리.

## 4.3 다중 라인 입력

`+ 품목 추가 / Thêm vật tư` 버튼으로 한 요청에 여러 품목을 함께 담을 수 있습니다. **모든 라인이 한 접수번호로 묶입니다** — 출고 담당자가 한 번의 출고 작업으로 처리합니다.

## 4.4 접수 후 일어나는 일

**요청 전송 / Gửi yêu cầu** 버튼 클릭 시:

1. **각 라인 호환성 검증** — 한 라인이라도 호환 외 품목이면 전체 거절(`not_compatible` + 해당 itemId 표시).
2. **접수번호 발급** — `YY/MM/DD-NN` 형식 (AS 와 동일 시퀀스 공유).
3. **티켓 생성** — `kind = SUPPLIES_REQUEST`, `status = RECEIVED`, 라인 정보가 `suppliesItems` JSON 컬럼에 저장.
4. **요약 메시지 자동 생성** — 「[소모품 요청] 토너 흑백 × 2, 잉크 컬러 × 1」 식으로 첫 화면에 노출.
5. **사내 알림** — Tellustech 출고 담당자에게 통지.
6. **메인 화면 갱신** — 「내 요청 현황」에 즉시 표시 (종류 컬럼에 「📦 소모품」 표기).

> 미수금 정지(`BLOCKED`) 시 AS 와 동일하게 빠른 카드가 비활성화됩니다.

# 5. 사용량 컨펌

IT(프린터·복합기) 월별 사용량(카운터)을 확인하고 청구 금액을 서명·컨펌합니다.

## 5.1 표시 데이터 — 어디서 오는가

화면에 보이는 각 줄은 사내 직원이 등록한 **월별 청구 행(`itMonthlyBilling`)** 1건입니다. 자동으로 본인 거래처의 IT 계약 청구만 노출됩니다.

| 컬럼 | 출처 / 의미 |
|---|---|
| **월 / Tháng** | `billingMonth` (예: `2026-04`) — Tellustech 담당자가 매월 등록 |
| **S/N** | 그 청구가 가리키는 IT 장비의 시리얼 |
| **흑백 / B&W** | `counterBw` — 사내 담당자가 검침해 입력한 누적값 |
| **컬러 / Color** | `counterColor` — 동일 |
| **금액 / Phí** | `computedAmount` — 시스템이 카운터 차이 × 단가로 자동 계산 |

## 5.2 입력값별 데이터 영향도

| 입력 / 동작 | **이 값이 시스템에서 하는 일** |
|---|---|
| **✍️ 서명** (canvas 그리기) | 손가락/마우스 궤적이 PNG (data URL) 로 변환되어 메모리에만 보관 — 아직 DB 저장 안 됨 |
| **확인 / Xác nhận** 버튼 | 서명 PNG 가 API 로 전송 → `itMonthlyBilling.customerSignature` 컬럼에 저장 → 그 청구 행이 잠김 (재서명 불가) → 사내 ERP 의 미수금 자동 발생 트리거 |

## 5.3 모바일 손가락 서명 절차

1. 청구 줄 우측의 **✍️ 서명 / Ký tên** 버튼을 누릅니다 → 서명 모달이 열림.
2. 모달 안의 흰색 영역에 마우스(또는 스마트폰 손가락)로 서명을 그립니다.
3. 모달의 **확인** 버튼으로 서명을 저장합니다. (모달 닫히고 줄에 ✓ 표시)
4. 표 줄의 **확인 / Xác nhận** 버튼을 누르면 서명이 서버로 전송되고 컨펌이 완료됩니다.

## 5.4 검증 규칙 (저장 시점)

| 검증 | 실패하면 |
|---|---|
| 본인 거래처의 청구 행만 컨펌 가능 | `forbidden` 에러 |
| 이미 서명된 행은 재서명 불가 | `already_confirmed` 에러 표시 |
| 서명 누락 시 「확인」 버튼 비활성화 | 클라이언트에서 차단 |

## 5.5 컨펌 후 일어나는 일

1. 청구 행이 잠김 (`customerSignature` not null).
2. 사내 ERP 미수금 모듈에서 그 청구액이 **새 미수금**(`PayableReceivable`, `kind=RECEIVABLE`, `status=OPEN`)으로 자동 생성.
3. 결제 만기일이 거래처 마스터의 결제 조건(예: 30일)으로 자동 계산.
4. 회계마감(잠금) 정책의 영향을 받기 시작.

## 5.6 정지(BLOCKED) 상태와 무관

사용량 컨펌은 미수금 정지 상태와 관계없이 항상 사용할 수 있습니다 — 컨펌 자체가 결제와는 별개의 「수치 확정」 절차이기 때문입니다.

> 한 번 컨펌하면 수정할 수 없습니다. 카운터 값에 이의가 있을 경우 **반드시 컨펌 전에** Tellustech 담당자에게 연락해 주세요.

# 6. 교정 성적서 다운로드

교정(검교정) 작업이 완료된 장비의 성적서 PDF 를 검색·다운로드합니다.

## 6.1 입력값별 데이터 영향도

각 검색 조건은 서버 쿼리에 부분 일치(`contains`) AND 조건으로 더해집니다. 모두 비워 두고 「조회」를 누르면 본인 거래처의 모든 성적서가 발행일 내림차순으로 표시됩니다.

| 조건 | 매칭 컬럼 | **검색 동작** |
|---|---|---|
| **S/N** | `SalesItem.serialNumber` | 부분 일치 (대소문자 무시) — 예: `SN-12` 입력 시 `SN-12345`, `ASN-12` 등 모두 매치 |
| **성적서번호** | `SalesItem.certNumber` | 부분 일치 |
| **품목** | `Item.itemCode` 또는 `Item.name` | 부분 일치 — 코드/이름 어느 쪽이든 |
| **시작일** | `SalesItem.issuedAt >= 시작일` | 발행일 하한 |
| **종료일** | `SalesItem.issuedAt <= 종료일` | 발행일 상한 |

## 6.2 어떤 행이 검색 대상인가 — 자동 필터

본 화면이 보여주는 데이터는 다음 4가지 조건이 동시에 만족되는 매출 라인입니다.

1. `Sales.clientId` = 본인 거래처 (자동 — 다른 거래처 데이터는 절대 안 보임)
2. `Sales.project.salesType` = `CALIBRATION` (교정 프로젝트만)
3. `SalesItem.certNumber` 또는 `SalesItem.certFileId` 중 **하나라도** 값이 있음
4. (위에 추가로) 화면 검색 조건이 매치

따라서 일반 매출(`TRADE` 등) 라인이나 PDF 가 아직 업로드 안 된 교정 라인은 노출되지 않습니다.

## 6.3 결과 표

| 컬럼 | 의미 |
|---|---|
| **발행일** | `SalesItem.issuedAt` |
| **성적서번호** | `SalesItem.certNumber` |
| **S/N** | `SalesItem.serialNumber` |
| **품목** | `itemCode · name` |
| **다음 기한** | `SalesItem.nextDueAt` — 다음 정기 검교정 권장일 |
| **다운로드** | 📄 PDF 버튼 — `certFileId` 가 있을 때만 표시 |

## 6.4 다운로드 동작

📄 PDF 버튼을 누르면 새 창에서 `/api/files/{certFileId}` 가 열립니다.
- 본인 거래처의 파일만 다운로드 가능 — 다른 거래처 ID 의 파일을 직접 URL 로 시도해도 `403` 거절.
- 파일이 열리면 브라우저의 인쇄·다운로드 메뉴로 로컬 저장 가능.
- 메인 화면 「📄 교정 성적서 ({N})」 의 N 값은 본 화면이 노출 가능한 총 건수와 같습니다.

# 7. 모바일 PWA 설치

포탈은 PWA(Progressive Web App) 로 동작하므로 스마트폰 홈 화면에 앱처럼 설치할 수 있습니다.

## 7.1 iOS Safari

1. Safari 로 포탈 주소에 접속해 로그인까지 완료합니다.
2. 하단 가운데 **공유** 버튼(□↑) 을 누릅니다.
3. **「홈 화면에 추가」** 를 선택합니다.
4. 이름을 확인하고 **추가** 를 누르면 홈 화면에 아이콘이 생성됩니다.

## 7.2 Android Chrome

1. Chrome 으로 포탈 주소에 접속해 로그인까지 완료합니다.
2. 우상단 **⋮ 메뉴 → 「홈 화면에 추가」** 를 선택합니다.
3. **설치** 를 누르면 앱 서랍·홈에 아이콘이 생성됩니다.

## 7.3 앱처럼 사용하기

- 설치 후에는 브라우저 주소창 없이 전체 화면으로 실행됩니다.
- 일부 화면(메인·내 요청 현황 등)은 오프라인에서도 마지막 본 데이터를 캐시로 표시합니다. 새 요청·실시간 상태 갱신은 인터넷 연결이 필요합니다.

# 8. FAQ

**Q. 비밀번호를 잊었습니다.**
A. Tellustech 담당자에게 연락해 재설정을 요청해 주세요. 자가 재설정은 제공하지 않습니다.

**Q. AS 요청 진행 상황은 어디서 봅니까?**
A. 메인 화면 「내 요청 현황」 표에서 4단계(요청 → 진행중 → 완료 → 확인됨)로 추적됩니다. 알림은 등록된 이메일·카카오로 함께 전송됩니다.

**Q. 소모품 목록에 원하는 품목이 보이지 않습니다.**
A. 호환 매핑이 등록되지 않았을 가능성이 큽니다. 담당자에게 해당 품목을 본인 장비에 매핑해 달라고 요청해 주세요.

**Q. 사용량 컨펌에서 카운터 값이 잘못된 것 같습니다.**
A. **컨펌 버튼을 누르기 전에** 담당자에게 연락해 주세요. 컨펌이 완료되면 수정할 수 없습니다.

**Q. 회사 헤더에 빨간 정지 배지가 떠 있습니다.**
A. 미수금 결제가 지연되어 신규 요청 메뉴가 일시 비활성화된 상태입니다. 결제 확인 후 자동 해제됩니다. 사용량 컨펌·교정 성적서 다운로드는 정지와 무관하게 사용 가능합니다.

**Q. 알림은 어디로 옵니까?**
A. 등록된 이메일 / 카카오톡으로 전송됩니다. 수신 채널 변경은 담당자에게 문의해 주세요.

**Q. 한국어와 베트남어를 동시에 보고 싶습니다.**
A. 입력 시에는 원문 언어 한 개만 선택해 작성하면 시스템이 자동으로 나머지 언어를 번역해 보관합니다. 표시 언어는 화면 우상단 국기 아이콘으로 전환할 수 있습니다.
